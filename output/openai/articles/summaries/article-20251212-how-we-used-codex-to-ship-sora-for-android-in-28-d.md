# How We Used Codex to Ship Sora for Android in 28 Days


## Summary
OpenAIは、Codexを活用してSora Androidアプリをわずか28日間で開発し、2025年11月にローンチしました。このアプリはローンチ初日にPlay Storeで1位を獲得し、24時間で100万本以上の動画がAndroidユーザーによって生成されました。この驚異的なスピードは、少人数のエンジニアチームがCodex（GPT-5.1-Codexモデルの初期バージョン、約50億トークンを消費）と緊密に連携したことで実現しました。記事では、Codexを新しく採用されたシニアエンジニアのように扱い、その能力を最大限に引き出すためのアプローチが詳細に説明されています。

Codexは、大規模なコードベースの迅速な理解、広範な単体テストの作成、フィードバックへの迅速な対応、複数のアイデアの並行実行、そして問題解決における新たな視点の提供といった点で優れた能力を発揮しました。これにより、人間はコードの記述に費やす時間を減らし、アプリのアーキテクチャ設計、ユーザー体験、システム全体の変更、そして最終的な品質管理といった、より戦略的で高レバレッジな作業に集中することができました。開発チームは、プロダクトの構造的決定とコーディング規範を設定し、Codexがその枠組み内で効率的にコードを生成できるよう導きました。

開発プロセスでは、Codexに明確な目標、制約、そして「我々のやり方」に関するガイダンスを与えることが重要でした。また、非自明な変更に対しては、Codexにまずシステムの仕組みを理解させ、堅牢な実装計画を立てさせてからコードを生成させるという「計画ループ」が有効でした。Sora Androidアプリの開発では、iOS版のコードベースやバックエンドの情報をCodexに参照させることで、SwiftからKotlinへのロジックの移植を効率化し、クロスプラットフォーム開発の新たな可能性を示しました。AIアシスト開発は厳密さの必要性を減少させるのではなく、むしろ増加させ、未来のソフトウェアエンジニアは深いシステム理解とAIとの協調作業が求められると結論付けています。

## Key Points
- Sora Androidアプリは、Codexの活用によりわずか28日で開発され、2025年11月のローンチ後24時間で100万本以上の動画が生成され、Play Storeで1位になった。
- 開発には、約50億トークンを消費したGPT-5.1-Codexモデルの初期バージョンが使用された。
- Codexは、大規模コードベースの理解、単体テスト作成、フィードバック適用、並行作業、新たな視点の提供、高レバレッジ作業の実現に優れている。
- 開発チームは、アプリのシステム設計、アーキテクチャ、ユーザー体験、主要なトレードオフ、最終的な品質管理に注力し、Codexに明確な指示とコンテキストを提供した。
- iOSコードベースをKotlinに効率的に翻訳することでクロスプラットフォーム開発を加速させ、Codexにとってコンテキストが極めて重要であることが示された。
